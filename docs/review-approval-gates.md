# Review Approval Gates

Review approval gates should help podcast teams move a finished episode from edit review to publishing without losing track of who still needs to sign off.

## User Goal

A creator or producer should be able to see which parts of an episode are approved, which still need review, and who owns the next decision before final export or publishing.

## Approval Areas

Track sign-off for the parts that commonly affect a long-form podcast release:

- host review
- guest name and link accuracy
- caption and glossary corrections
- sponsor placement and disclosure
- thumbnail or cover frame
- episode title and description
- chapter markers
- destination-specific publish settings
- final export package

Each approval area should link back to the exact workspace view where the issue can be checked or fixed.

## Review States

Use simple team-facing states:

- not requested
- waiting
- changes requested
- approved
- skipped

The product should show who changed the state, when it changed, and any short note left with the decision.

## Publishing Gate

Before final export or direct publishing, the product should make unresolved approval work visible without blocking every show in the same way.

Creators should be able to:

- require approvals for client or agency workflows
- skip approval areas that do not apply to the episode
- see changes requested in one compact list
- jump from an approval item to the relevant timeline moment or metadata field
- include approval status in the handoff summary

## Maintainer Acceptance Notes

Accept work that helps teams publish with confidence and clear ownership. Close work that turns approvals into generic project management, hides who requested changes, or blocks simple solo-host publishing flows with unnecessary ceremony.
