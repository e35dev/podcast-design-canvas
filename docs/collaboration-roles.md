# Collaboration Roles

Collaboration should give each reviewer or teammate the controls they need without exposing every production decision to everyone.

## User Goal

A creator should be able to invite collaborators to review, approve, or edit the right parts of a podcast episode.

## Roles

Support clear roles:

- owner
- producer
- editor
- host
- guest reviewer
- client approver
- sponsor reviewer

Each role should have a plain-language description before the invite is sent.

## Permissions

Permissions should map to podcast tasks:

- edit canvas layout
- approve b-roll
- correct captions
- review guest information
- approve sponsor placement
- edit metadata
- create review copy
- export final episode

Avoid generic permission names that do not describe what the collaborator can do.

## Review Experience

Collaborators should land on a focused view:

- assigned moments
- pending decisions
- comments addressed to them
- final approval state
- export impact of unresolved items

## Maintainer Acceptance Notes

Accept work that makes podcast production collaboration clearer for teams, agencies, guests, clients, and sponsors. Close work that adds generic project permissions without tying them to episode decisions.
